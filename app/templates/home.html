Internal Tool of Mushroom Edibility Feature Project


Welcome to the mushroom edibility DASHBOARD! This dashboard has all necessary data
that serves as a proof of concept for the proposed feature within the HikingPal App.
The feature will use a collected mushroom dataset that will be used as the base
of a trained machine-learning model. Using the model we'll be able to predict if the
is edible or not.



Descriptive and Non-Descriptive Methods

The tool includes two data methods:
* Descriptive method: three visuals that describe the data and supports the understanding of the data.
* Non-Descriptive method: based on the data we train our machine-learning model with
a Random Forest Classifier algorithm. The trained model then serves to predict the
edibility of a specific mushroom using its attributes.

Data Parsing

The Data source is in CSV format including characters describing the mushroom attributes.
In order to train our model, the data has to be read by a python csv parser, and the characters have to be
converted to numerical values. This project uses the ASCII equivalent of each character for simplicity.

Data Exploration and Visualization

By using three different visualizations of the data we are able to understand which attributes have a
key role in classifying the mushroom.
Attribute Importance: Using the Random Forest Classifier's feature importance method we can
can use a matplotlib graph to understand which are the key factors.
Correlation Heatmap: Using the correlation method of the data frame we can use a heatmap from
the seaborn library. We can see that there is a strong correlation e.g. for the gill-size and gill-color
the veil-type has no effect on other data as it is a constant value.
K-Means Cluster: Using the importance score we can pick the two most important attributes
which could be further analyzed by plotting them with K-Means clustering. We can then analyze
the centroids of the relevant attributes.

Interactive Queries

As a non-descriptive method, the dashboard is capable of interactively describing a mushroom by its
attributes and using the trained machine learning model we can predict if the mushroom is edible or not.

Machine-learning Methods and Algorithms

Our model is using Random Forest Classifier algorithm from the sklearn python library.
The application uses the corr() function the verify the correlation between all attributes.

Evaluating Data Accuracy

The application takes advantage of the accuracy_score function of the algorithm to split the data into
two sets. The train set will be used to train the model and the test set will verify the model's accuracy.
After using 30% of the data for testing we have resulted in an 83% accuracy.

Security Measures

The source code of the app is version-controlled on github with limited access exclusively
for developers. The main data frame can only be manipulated by changing the version-controlled
source code and adding additional lines to the csv file. The changes then will be reviewed
by other developers and only added to the production code if approved.

Product Health Monitoring

The app will be hosted by Heroku and the code will be deployed to its server. The Heroku web client
is capable of showing metrics and logs that serve to check if the tool is healthy. In case the app
crashes, an alert will be sent to developers and a server restart will automatically take place.
